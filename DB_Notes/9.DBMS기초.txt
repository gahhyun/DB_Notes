1. DBMS
    1) 데이터베이스를 관리하는 시스템
    2) 데이터를 저장하고 유지보수(수정, 삭제, 추가)하고 이를 검색하는 시스템
        -CAUD
    3) 대량의 데이터를 처리하는 시스템
    4) 다양한 자료구조와 검색구조 사용해 '빠른' 검색 가능
    5) 대부분의 시스템R(검색) >>>> CUD(업데이트)의 빈도수가 많음
    6) 검색에 최적화

2. RDBMS
    1) 관계형(Relational) 데이터베이스 시스템
    2) 테이블(Table based) 기반의 DBMS
        -테이블 -컬럼 형태의 데이터 저장 방식
        -테이블과 테이블 간의 연관관계(주로 외래키형태)을 이용해 필요한 정보를 구하는 방식
    3) 모델링은 E-R (Entity Relationship) 모델을 사용
    4) 테이블을 엔터티(기본)와 릴레이션십(유도) 테이블로 구분하는 방식
    5) 데이터를 테이블 단위로 관리
        -하나의 테이블은 여러 개의 컬럼으로 구성됨
    6) 테이블끼리의 중복정보는 최소화시킴
        -정규화 => 정규형
    7) 여러 테이블을 합쳐 큰 테이블을 생성(JOIN)해서 필요한 정보를 찾아내는 방식


3. 기본용어
    1) 스키마(Schema)
        -DB, 테이블 정의 내역
        -데이터베이스에 저장되는 데이터의 논리적구조와 제약조건을 정의한 것
    2) SQL 쿼리 (SQL Query)
        -관계형 DBMS 사용하는 전용 질의언어
    3) 기본키 (Primary Key : PK)
        -테이블에서 하나의 레코드를 지정할 수 있는 하나 이상의 컬럼집합
    4) 외래키 (Foreign Key : FK)
        -어떤 테이블의 기본키가 다른 테이블의 컬럼에 들어 있는 경우
    5) 레코드 (Record)
        -테이블에 들어 있는 여러가지 인스턴스 하나하나를 지정
        -기본키(PK)로 구별가능

4. SQL (Structured Query Lenguage)
    1) 데이터베이스에 있는 필요한 정보를 사용할 수 있도록 도와주는 언어
    2) 인터프리터(컴파일x)
    3) 대소문자를 구분하지 않음 (데이터 내용은 구별함)
    4) 종류
        -DDL : Data Definition Language
            -DB, 테이블의 스키마를 정의, 수정하는 기능
            -테이블 생성, 컬럼 추가, 타입변경, 각종 제약조건 지정, 수정 등
        -DML : Data Manipulation Language
            -테이블의 데이터를 조작하는 기능
            -테이블의 레코드를 CRUD(Create,Retrieve,Update,Delete)


5. 데이터 타입
    1) 문자형
        -CHAR(길이)
            -고정길이 문자열을 저장함
            -예) CHAR(5) : 4자리 문자열 + 공백 , 문자열 다 사용함
        -VARCHAR(길이)
            -가변길이 문자열을 저장함
            -예) VARCHAR(5) : 4자리 문자열 끝 , 데이터의 길이만큼만 사용함
        -TEXT
            -길이제한이 없는 가변길이 문자열 저장

    2) 숫자형
        -NUMBER(총자릿수, 소수점자릿수)
            -예) NUMBER(7,3)은 9999.999까지 저장가능
        -NUMERIC(총자릿수)
            -예) NUMERIC(7)은 최대 9999999까지 저장 가능
        -SMALLINT         : 정수형
        -INTEGER 혹은 INT : 정수형
        -BIGINT           : 정수형(성능저하)
        -SERIAL           : 데이터가 저장될때마다 순차적으로 유일한 숫자(번호)가 저장됨

    3) 날짜형
        -DATE
            -날짜값을 저장함
                -YYYY-MM-DD (2023-03-02)
        -TIME
            -시간값을 저장함
            -시간값은 최대 6자리 정밀도를 가짐
        -TIMESTAMP
            -시간대(TIMEZONE)가 없는 년월일시분초.마이크로세컨드를 저장함
        -TIMESTAMPZ
            -시간대(TIMEZONE)가 있는 년월일시분초.마이크로세컨드를 저장가능
                -예) 영국시간, 미국시간, 한국시간
