1. 요구사항 분석
    이젠 도서관은 많은 책을 보유하고 있으며, 책을 대여하는 시스템을 개발 중이다
    => 도서 관리 시스템을 개발 중이다

    우리는 많은 회원을 관리하고 있다 회원의 이름, 연락처, 주소를 관리한다
    => #회원 엔터티가 도출된다
        속성으로는 회원번호, 회원명, 연락처, 주소를 가진다

    대여한 책을 3회 이상 기한 내에 반납하지 않은 회원은 비신뢰회원으로 등록한다
    => 회원 엔터티에 비신뢰회원여부 속성을 추가한다

    비신뢰회원이 되면 1년 동안 책을 대여해 주지 않는 패널티 조항이 있다
    따라서 회원의 비신뢰 여부를 관리하며, 비신뢰회원일 경우 최종 비신뢰 회원 등록일자를 관리한다
    => 회원 엔터티에 비신뢰회원등록일자 속성을 추가한다

    우리는 많은 책을 보여하고 있다 도서명, 출판사, 출판일자, 이젠도서관 납품일자를 관리한다
    => #도서 엔터티가 도출된다
        속성으로는 도서번호, 도서명, 출판사를 가진다
    => #보유도서 엔터티가 도출된다
        속성으로는 보유도서번호, 도서번호, 납품일자를 가진다

    회원은 도서를 대여할 수 있으며, 한번에 여러 권을 대여할 수 있다
                ----             --------------------
    또한 도서별로 반납예정일자가 다를 수 있다 회원의 실제 반납은 반납예정일과 다를 수 있어 별도로 반납일자를 관리해야 한다
                                                            ---------
    도서대여일자, 대여기간을 관리해야 한다
    도서대여는 한번 대여할 때 마다 대여번호가 자동으로 번호가 부여한다
    => #대여 엔터티가 도출된다
        속성으로는 대여번호, 대여일자, 대여기간, 회원번호(FK), 대여료금액이 있다
    => #대여상세 엔터티가 도출된다
        속성으로는 대여상세번호, 대여번호(FK), 고유도서번호, 반납예정일자, 실제반납일자, 연체여부, 연체료금액이 있다

    우리는 도서관 운영비용을 회원의 회비와 도서대여료, 도서반납 연체료로 운영한다
    => #회비 엔터티가 도출된다
        속성으로는 회비번호, 회원번호(FK),

    회원의 회비는 년간 1회 비용을 납입하며, 도서대여비는 대여할 때 일시불로 받는다
    도서반납연체료는 도서를 반납하는 시점에 받는다

3. 데이터 모델링 작업 수행 (DA# Modeler5)
    1) 특정 도서는 하나 혹은 여러개의 보유도서를 가질 수 있다
           ----                    --------
    2) 특정 회원은 하나 혹은 여러번에 걸쳐서 대여 할 수 있다
            ---                           ---
    3) 하나의 대여는 하나 혹은 여러개의 대여상세를 가질 수 있다
             ----                    --------
    4) 보유도서는 하나 혹은 여려개의 대여상세를 가질 수 있다
       -------                    --------
    5) 회원은 여러번에 걸쳐서 회비를 낼 수 있다
       ---                  ---
    6) 입금유형으로는 회비입금, 대여입금, 연체입금이 있다
       -------      --------  -------  -------
    7) 특정 대여상세는 하나의 입금내역을 가질 수 있다
            -------         -------
    8) 특정 대여는 하나의 입금내역을 가질 수 있다
            ---         --------
    9) 특정 회비는 하나의 입금내역을 가질 수 있다
            ---         --------
    10) 회원이 비신뢰회원이 되면 그제서야 비로서 비신뢰회원등록일지가 저장된다
        ---                                  -----------------
    11) 대여상세에서 반납이 완료되면 그제서야 비로소 반납일지가 저장된다
        -------                                  -------
    12) 대여상세에서 연체가 발생되면 그제서야 연체료금액이 저장된다
        -------                           ----------
